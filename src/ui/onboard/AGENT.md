# Agent Context Anchor - src/ui/common/setup

본 문서는 `src/ui/common/setup` 도메인의 컴포넌트 및 스타일 코드에 대한 에이전트 전용 컨텍스트 앵커입니다. 에이전트는 이 디렉토리 내의 파일들을 생성, 수정, 리팩토링할 때 아래의 규칙을 반드시 준수해야 합니다.

---

## 1. 디렉토리 구조 및 핵심 역할

```text
setup/
├── AGENT.md                 # 본 문서 (에이전트 제어용)
├── index.js                 # setup 도메인 배럴 파일 (외부 노출 제어)
└── [Component]/             # 온보딩 단계별 격리 폴더
    ├── index.js             # 배럴 파일
    ├── [Component].jsx      # 리액트 비즈니스 로직 및 마크업
    └── [Component].styles.js # 격리된 styled-components 선언
```

### 각 컴포넌트의 역할
- **Setup**: 신규 사용자 온보딩을 총괄하는 컨테이너 컴포넌트입니다. 라우팅 흐름을 중재하며 `Suspense` 폴백 영역을 지원합니다.
- **Nick**: 사용자가 MyVoca를 이용하기에 앞서 개성 있는 닉네임을 생성하도록 입력 필드를 바인딩하고, 완료 후 로컬 스토리지에 이를 저장하는 닉네임 설정 단계입니다.
- **Data**: 닉네임 작성이 끝난 후, 초급/중급/고급의 난이도별 학습 단어장을 선택하게 하여 personal Voca 데이터셋을 빌드하고 로드하는 데이터 생성 단계입니다.

---

## 2. 참조 및 의존성 불변 규칙 (Crucial Rules)

1. **상대 참조 금지 (상위 디렉토리 참조 시 절대 경로 사용)**
   - 공용 컴포넌트나 레이아웃, 외부 도메인을 참조할 때는 어떠한 경우에도 `../../common/...` 과 같은 복잡한 상대 경로를 사용할 수 없습니다.
   - 반드시 **절대 경로 별칭 (`@/ui/common/...`, `@/common/api/...`)**을 사용하여 모듈 참조 오류를 차단하십시오.
2. **스타일 파일의 단방향 의존성**
   - `*.styles.js` 파일은 컴포넌트 로직(`*.jsx`) 내에 선언된 훅이나 일반 자바스크립트 변수를 절대 임포트할 수 없습니다.
   - 스타일링에 필요한 가변 속성은 오직 리액트 프롭스(`props`) 및 스타일 전용 임시 프롭(`$prefix` 형식)을 통해서만 전달받아야 합니다.
3. **진입점 최소화 (배럴 패턴)**
   - 도메인 최상위 `index.js`에 명시적으로 노출된 컴포넌트만 외부에서 사용할 수 있습니다. 외부 폴더에서 이 도메인의 서브컴포넌트를 임포트할 때는 반드시 `import { Setup, Nick, Data } from "@/ui/common/Setup"` 형태로만 임포트해야 합니다.
