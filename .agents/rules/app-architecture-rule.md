# 공통 아키텍처 금지 규칙 (app-architecture-rule)

본 규칙은 프로젝트 오염 방지를 위한 절대적 제약 조건이다. 에이전트는 코딩 시 본 규칙을 단 1글자도 위반해서는 안 된다.

## 1. [금지] 의존성 역방향 임포트 금지
- `src/common`, `src/api` 등 하위 비즈니스 레이어는 상위 레이어인 `src/components` (또는 `src/ui`) 내부 파일 및 리액트 컴포넌트를 절대로 임포트(import)할 수 없다.
- **하이브리드 환경 제약**: Next.js 서버 컴포넌트(Server Component) 또는 순수 서버 사이드 레이어는 클라이언트 컴포넌트 전용 자산(useState 기반 커스텀 훅, `styled-components` 등의 런타임 CSS-in-JS 스타일 등)을 역방향으로 상향 임포트할 수 없다.

## 2. [금지] 순환 참조 금지
- 파일 간 서로가 서로를 상호 호출하거나 가져다 쓰는 어떠한 형태의 의존성 순환 고리(Circular Dependency) 생성도 절대 금지한다.

## 3. [금지] 커스텀 훅 무단 방치 및 오염 금지
- 특정 컴포넌트/도메인 전용 훅은 반드시 해당 컴포넌트 디렉토리 내부에 위치해야 하며, 공통 폴더로 함부로 승격시켜 아키텍처를 오염시키는 것을 금지한다.
- 반대로, 앱 전역에서 사용하는 공용 훅은 개별 컴포넌트 디렉토리 내에 방치해서는 안 되며 오직 `src/common` 하위(또는 전역 `store` 내부)로만 한정하여 관리한다.

## 4. [금지] API 단일 파일 몰아쓰기 금지
- `src/api` 하위에 모든 도메인 API 호출 함수를 단일 거대 파일에 몰아넣어 작성하는 행위를 금지한다. 반드시 `src/api/[:데이터의 종류]/`와 같이 도메인별 폴더 격리를 철저히 엄수한다.
