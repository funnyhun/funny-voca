# 학습 및 퀴즈 플레이어 시나리오 명세서 (Study Play & Quiz Player Scenario)

본 문서는 MyVoca 애플리케이션의 핵심 학습 시나리오인 암기 카드 모드, 실시간 타이머 퀴즈 모드, 그리고 완료 통계 연동 및 동적 경로 제어 메커니즘을 상세히 정의합니다.

---

## 1. Play & Quiz 주소 완전 은닉화 (URL Parameter Concealment)

학습 집중도를 저하시키는 복잡한 하위 경로 노출 및 뒤로가기 오작동 문제를 예방하기 위해, 모든 학습 모드는 파라미터가 배제된 **단일 고정 경로**로 동작합니다.

- **학습용 고정 URL**: `/play`
- **퀴즈용 고정 URL**: `/quiz`

### 1.1 동적 데이터 마운트 프로세스
- 리액트 라우터를 통한 리다이렉트나 파라미터 딥 링킹 기법을 원천 배제합니다.
- 사용자가 `/play` 또는 `/quiz` 경로에 안착하면, 플레이어 컴포넌트가 마운트되는 즉시 시스템 전역 상태 프로필 캐시(`KEYS.PROFILE`) 내에 저장되어 있던 **`selected` (현재 설정된 고유 청크 ID)**를 조용히 불러옵니다.
- 이에 해당하는 단어 뭉치를 로컬 사전 캐시(`KEYS.MASTER`)로부터 O(1) 성능으로 동적 추출하여 플레이 상태에 즉각 바인딩합니다.
- 이로 인해 URL 복사나 파라미터 수동 변조를 통한 비정상 데이터 로드 오류를 원천 차단하고 단일 통합 진입점을 보장합니다.

---

## 2. 학습 모드 라이프사이클 흐름 (Study Lifecycle Flow)

학습 시나리오는 크게 **1단계: 암기 카드 모드**와 **2단계: 문장 빈칸 퀴즈 모드**의 단계적인 유기적 결합으로 흐릅니다.

```mermaid
graph TD
    Dashboard[홈 화면 / PlayButton 클릭] --> |1. selected 청크 로드| PlayEntry[/play 진입]
    PlayEntry --> CardStage[2. 카드 모드: 영어 스펠링 노출]
    CardStage --> |카드 탭 Flip| CardBack[뜻, 발음기호, 예문 학습]
    CardBack --> |다음 단어 슬라이드| CheckCardEnd{40개 단어 카드 모두 완료?}
    
    CheckCardEnd -- "아니오" --> CardStage
    CheckCardEnd -- "예 (카드 완료)" --> QuizEntry[/quiz 강제 전이]
    
    QuizEntry --> QuizStage[3. 퀴즈 모드: CircleTimer 카운트다운 시작]
    QuizStage --> |객관식/주관식 풀이| UserSubmit[사용자 정답 제출]
    UserSubmit --> |정오답 및 오디오 피드백| CheckQuizEnd{40개 단어 퀴즈 모두 완료?}
    
    CheckQuizEnd -- "아니오" --> QuizStage
    CheckQuizEnd -- "예 (퀴즈 완료)" --> CompleteStage[4. Complete: 최종 결과 화면]
    
    CompleteStage --> |voca.status=true 및 completed_at 마킹| UpdateScore[Streak 가산 및 selected 다음 전진]
    UpdateScore --> Exit[홈으로 이동 / 다음 청크 연속 학습 선택]
```

## 3. 세부 학습 단계별 비즈니스 로직

### 3.1 암기 카드 모드 (Card Mode)
- **핵심 역할**: 인지 과부하 없는 직관적인 플래시 카드 암기 단계입니다.
- **주요 UI/UX 로직**:
  - 카드 앞면에는 순수 영어 스펠링과 발음 청취 아이콘만 노출하여 인지 자극을 극대화합니다.
  - 카드를 탭하는 순간 마이크로 애니메이션과 함께 카드가 뒤집히며 품사, 상세 한글 뜻, 영문/국문 매칭 예문이 글래스모피즘 오버레이 내에 유려하게 노출됩니다.
  - 사용자는 각 카드의 학습을 끝마치면 "다음 단어" 버튼을 통해 넌블로킹 슬라이드로 넘어갑니다.

### 3.2 퀴즈 풀이 모드 (Quiz Mode)
- **핵심 역할**: 카드 모드에서 습득한 어휘의 실제 예문 문맥 적용 능력을 확인하는 몰입형 테스트 단계입니다.
- **주요 비즈니스 로직**:
  - **서클 타이머 (`CircleTimer`)**: 문항당 할당된 제한 시간 동안 실시간 게이지 바가 수축하여 긴장감과 몰입감을 배가합니다. 제한 시간 초과 시 오답으로 즉각 처리됩니다.
  - **빈칸 퀴즈 포맷**: 단어 데이터 내의 `quiz_en` 및 `quiz_ko` 템플릿을 읽어와, 핵심 키워드가 빈칸(`___`)으로 숨겨진 고품질 예문 기반 객관식/주관식 문항을 동적 출제합니다.
  - **즉시 피드백**: 제출 즉시 마이크로 오디오 이펙트와 함께 정오답 인터페이스가 활성화되어 오답을 즉각 보정하도록 안내합니다.

### 3.3 학습 완료 모니터링 및 커밋 (`Complete`)
- 퀴즈의 최종 3단계를 완료하면 세션 통계 결과(오답 없이 맞춘 단어 개수, 걸린 시간 등)를 산출합니다.
- **최종 커밋 트리거**: 기존에 `Complete.jsx`에서 호출되던 이중 갱신 방식을 탈피하고, **`Quiz.jsx`에서 3단계의 마지막 문제 완료 즉시 단 1회의 원샷 API(`updateVoca`) 호출을 통해 원자적(Atomic)으로 완료 처리를 수행**합니다.
- **성취 판별 및 동작 분기**:
  - **100% 무결점 완수 (`done.length === word.length`)**: 대상 청크의 상태를 완벽한 완료(`status = true`, `completed_at = YYYY-MM-DD`)로 봉인하고, 당일 최초 완료인 경우 연속 학습(Streak) 가산 및 다음 추천 청크로 `selected`를 전진시킵니다.
  - **오답 발생 시 (`done.length < word.length`)**: 청크 완료를 커밋하지 않고 `status = false`, `completed_at = null`로 유지하며, Streak 가산 및 selected 전진 없이 제자리에 둡니다.
- **Complete.jsx 역할**: 이중 API 중복 갱신을 수행하지 않고, `Quiz.jsx`에 의해 이미 갱신된 최신 로컬 캐시를 읽어와 완벽 마스터 여부 및 당일 최초/추가 완료 여부에 따른 극적인 UI 및 격려 멘트 분기 연출을 수행하는 데에만 전념합니다.
