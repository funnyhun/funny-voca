# React SPA 및 클라이언트 컴포넌트 특화 금지 규칙 (react-expert-rule)

본 규칙은 React(Vite) 및 Next.js 클라이언트 환경 하위에서 에이전트가 절대로 저질러서는 안 되는 치명적 금지 조항이며, 1글자의 위반도 허용하지 않는다.

## 1. [금지] 2선 아토믹 컴포넌트 격리 규격 위반 금지
- 모든 UI 컴포넌트는 단일 파일에 스타일 객체와 컴포넌트 구조를 모조리 뭉뚱그려 작성해서는 안 되며, 반드시 `index.tsx` (본체 컴포넌트 및 로직) + `[ComponentName].style.ts` 또는 `style.ts` (인라인 CSSProperties 스타일 컬렉션 격리)의 **2선 아토믹 컴포넌트 규격**을 엄격히 준수한다.
- 스타일 코드를 `index.tsx` 내부에 직접 하드코딩하거나 무질서하게 섞어 쓰는 행위를 절대 금지하며, 반드시 격리된 스타일 파일로부터 `styles` 객체를 임포트하여 `as React.CSSProperties` 타입 단언을 입혀 인라인 스타일로 바인딩해야 한다.

## 2. [금지] 디자인 시스템 파편화 및 개별 .pen 파일 무단 생성 금지
- 개별 컴포넌트 디렉터리 내에 독립적인 `.pen` 메타데이터 파일을 무단으로 생성하는 행위를 절대 금지한다.
- 모든 디자인 스펙, 가이드라인 및 상태 정의는 반드시 **[funny-todo.pen](file:///z:/home/minhulee/Projects/funny-todo/.agents/funny-todo.pen)** 단일 통합 디자인 프로젝트 파일 내에 병합하여 통합 관리해야 한다.

## 3. [금지] Next.js 런타임 지시어("use client") 누락 금지
- Next.js App Router 환경 하위에 컴포넌트가 조립될 때, 해당 컴포넌트나 하위 파일이 리액트 커스텀 훅이나 런타임 CSS-in-JS 스타일을 임포트하고 있다면 본체 파일 최상단에 `"use client"` 지시어 선언을 누락하는 행위를 절대 금지한다.
 
## 4. [금지] useEffect 기반 비동기 데이터 사전 패칭 금지
- 화면 진입 시 필요한 데이터를 불러오기 위해 화면/컴포넌트 내부의 `useEffect` 훅 내에서 API를 직접 호출하는 행위를 엄금한다.
- **SPA 환경**: 반드시 `react-router`의 `Loader` 기능을 통해 사전 로딩(Pre-fetching)을 수행한다.
- **Next.js 환경**: 반드시 상위 서버 컴포넌트(`page.tsx`)로부터 사전 패칭이 완료되어 전달된 깨끗한 Props 데이터를 즉시 주입받아 구조분해 할당으로 사용하는 흐름을 준수한다.
 
## 5. [금지] 라우팅 로더 단일 파일 통합 금지 (SPA 한정)
- `router/loader/` 하위에 도메인 격리 없이 하나의 거대한 단일 파일에 모든 라우팅 로더를 쑤셔넣어 선언하는 행위를 금지한다. 반드시 `router/loader/[:데이터의 종류]/` 디렉토리 분류를 엄격히 준수한다.
