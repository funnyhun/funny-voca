# 시나리오 기반 개발 및 명세 전문가 (Scenario-Based Development Expert)

MyVoca 프로젝트에서 소스 코드 개발이나 설계 명세 수정을 수행할 때, 다음의 시나리오 중심 개발 프로세스와 문서 작성 표준 가이드라인을 엄격히 준수하여 수행해야 합니다.

---

## 1. 시나리오 중심 비즈니스 개발 프로세스 (Scenario-Driven Development)

새로운 코드를 구현하거나 기능을 추가할 때, 요구사항 명세만을 보고 기술적으로 접근하지 마십시오. 반드시 **사용자 시나리오**를 출발점으로 정의하고 구현을 도출하는 3단계를 수행하십시오.

1. **시나리오 정의 (Scenario)**
   - 학습자가 겪는 구체적인 일반적 비즈니스 상황과 목적을 사용자 관점의 행동 흐름으로 명확히 정의합니다.
   - 기술적인 구현 용어(예: JSON, DB 테이블, 쿼리, 큐 등)는 시나리오 단계에서 서술하지 마십시오.
2. **필수 기능 & UX 요건 명세 (UX Requirements)**
   - 해당 시나리오를 만족하기 위해 시스템과 화면 UI가 보장해야 하는 기능적 결과물과 사용성 경험(0ms 반응성, 진짜 진행률 바 등)을 정의합니다.
3. **구현 기술 및 데이터 설계 (Tech & Data)**
   - 위의 사용성(UX) 요건을 완벽하게 만족시키기 위해 생겨난 특수한 기술적 해결책(예: lte 범위 쿼리, 3단계 치환, WordQueueManager의 틱 제어 등)과 구체적인 소스코드 파일, 데이터 구조 매핑을 기술합니다.

## 2. 설계 명세 문서 작성 표준 가이드라인

`.agents/` 디렉토리 내의 명세서를 작성하거나 보완할 때는 다음 세부 규칙을 엄격히 지켜 문서의 무결성과 응집도를 유지해야 합니다.

### 2.1 문서 최상단 YAML Frontmatter 규격
모든 설계 마크다운 문서 최상단에는 문서 속성을 파악할 수 있는 YAML 헤더를 필수로 포함해야 합니다.
```yaml
---
category: "domain"      # core, domain, etc 중 하나만 사용
description: "학습 난이도 변경 및 스케줄 재배정 시의 데이터 제공 규칙과 시나리오 명세"
---
```

### 2.2 도메인 격리 폴더 및 3단계 문서 템플릿
- 도메인 문서들은 루트에 두지 말고 개별 폴더(`.agents/[도메인명]/[도메인명].md`)로 격리 배치하여 Depth를 최소화하고 응집합니다.
- 본문은 반드시 `## 1. 시나리오 정의`, `## 2. 필수 기능 & UX 정의`, `## 3. 구현 기술 및 데이터 매핑` 3개의 헤더로 구성합니다.

### 2.3 데이터 중복 서술 금지 및 schema.md 참조 무결성
- 각 도메인 문서에 데이터베이스 테이블 구조(컬럼 스펙)나 로컬 스토리지 캐시 스키마(JSON 등)를 중복하여 하드코딩해서 서술하는 행위를 엄격히 금지합니다.
- 데이터 설명이 필요한 경우, 직접 정의하지 말고 루트의 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md)를 마크다운 절대 경로 링크(`[schema.md](file:///...)`)로 기재하여 참조하도록 유도하십시오.
- 이를 통해 단일 신뢰 원천(Source of Truth)을 유지하고 정합성 유실을 방지합니다.

### 2.4 올바른 기술 용어 정립
- 마크다운 문서 내에서 실체가 모호한 '로컬 퍼스트(Local-First)'라는 표현을 남발하지 마십시오.
- 아키텍처를 정확히 반영하는 **'낙관적 업데이트 + 비동기 백그라운드 서버 저장'** 기법으로 기술 명세를 상세히 명문화하십시오.

## 3. 자동 검증 스크립트 실행 규칙 (Linter Execution)

작업 완료 전, 명세 파일들이 표준을 충족하는지 검사하는 자동화 파이썬 린터(`lint_specs.py`)를 무조건 구동하여 검증을 거쳐야 합니다.

- **실행 명령어**:
  `wsl --cd /home/minhulee/Projects/funny-voca/funny-voca-app/.agents/skills/scenario-based-develop-expert/scripts python3 lint_specs.py`
- **행동 강령**:
  - 린터를 돌려 오류 스트림이나 종료 코드 `1`이 발생하는 경우, 작업을 중단하고 출력된 규칙 위반 내용을 스스로 분석하여 해당 마크다운 문서를 수정한 뒤 린터를 다시 돌려 통과(종료 코드 `0`)시켜야 합니다.
  - 린터 검증을 완벽히 통과할 때에만 비로소 작업 완료를 보고하고 사용자에게 제출하십시오.
