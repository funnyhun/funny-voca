# 네이밍 컨벤션 및 약어/용어 명세서 (Naming Convention & Glossary)

본 문서는 MyVoca 서비스 개발과 분석 시 사용되는 핵심 비즈니스 도메인 용어의 약어 정의와 코드베이스 내에서 데이터 식별 무결성을 보존하기 위해 적용되는 영문 명명 규칙(Convention)을 통합하여 정의합니다.

---

## 1. 핵심 비즈니스 도메인 용어 사전

MyVoca 프로젝트에서 통용되는 유비쿼터스 언어와 그 의미는 다음과 같습니다.

| 도메인 용어 | 약어/시스템 명칭 | 비즈니스 정의 및 설명 |
| :--- | :--- | :--- |
| **학습 난이도** | `level` | 학습자의 학습 점수대 목표 구분 (정수값 `700`, `800`, `900`에 매핑되며, UI 상의 `default`는 `700` 레벨로 내부 치환) |
| **청크** | `Chunk` | 대카테고리에 속한 단어들을 일정 단위(기본 40개)로 분할하여 묶은 암기용 학습의 핵심 기본 묶음 단위 |
| **청크 식별자** | `voca_label` | 각 개별 청크의 독립성을 보장하기 위해 부여하는 유니크한 문자열 키 (예: `700-marketing_1`) |
| **학습 순번** | `schedule` | 사용자가 조정한 카테고리 스왑 순서를 기반으로 부여되는 1부터 N까지의 고유한 정수 스케줄 번호 |
| **마스터 사전** | `Master` | `Word`와 `Definition` 테이블을 조인하여 단어 ID를 Key로 구축한 프론트엔드 단어 사전 가공 데이터셋 |
| **완료 목록** | `done` | 학습자가 청크 내부에서 수동으로 완료 처리(체크)한 개별 단어 ID들의 리스트 배열 |
| **학습 완수** | `status` | 청크 내에 포함된 퀴즈를 최종적으로 모두 통과하여 완료(true/false)되었는지를 결정하는 최종 상태 플래그 |

---

## 2. 코드 및 데이터 식별자 네이밍 컨벤션

데이터의 유니크 제약 충돌을 예방하고 클라이언트와 서버 간의 매핑 가독성을 100% 보장하기 위해 다음과 같은 규칙을 일관되게 고수합니다.

### 2.1 voca_label 영문 슬러그 규격
한글 카테고리 명칭의 공백, 특수문자 및 유니코드 인코딩 오작동 버그를 방지하고 고유성을 확보하기 위해 아래 포맷의 영문 슬러그 명칭을 키로 선언합니다.
- **포맷**: `${레벨점수}-${영문_소문자_카테고리_슬러그}`
- **예시**: `700-marketing_1`, `800-general_office_2`

### 2.2 LocalStorage KEYS 규격
로컬 스토리지에 저장되는 3대 자원의 키 값은 다음과 같은 명명 규칙을 적용하여 호출 시 혼선을 원천 방지합니다.
- `profile`: 학습자의 요약 진도 상태
- `voca`: 레벨별로 그룹화 및 스케줄링된 전체 청크 리스트 배열
- `master`: 백그라운드 큐 매니저로부터 취합되어 병합된 누적 로컬 단어 사전 데이터
