# 학습 및 성취 도메인 명세서 (Study Play & Achievement Domain Specification)

본 문서는 학습자가 개별 청크 단어들을 플래시 카드로 암기하고, 최종적으로 퀴즈를 통과하여 하루 학습 목표를 완수하기 위한 사용자 중심 시나리오를 정의하며, 이를 완수하기 위해 설계된 시스템 기능 및 구체적인 기술 매핑 사양을 명세합니다.

---

## 1. 일반 학습 시나리오 (Vocabulary Card Memorization)

### 1.1 시나리오 정의
- **상황**: 학습자가 대시보드에서 "암기하기" 버튼을 눌러 단어 카드 학습 화면(`/play`)으로 진입하는 시나리오입니다. 학습자는 영단어 카드들을 좌우로 넘겨가며 아는 단어와 모르는 단어를 식별하고 완료 토글(체크) 처리를 직관적으로 기록합니다.
- **사용자 흐름**: 암기하기 클릭 -> 단어 카드 화면 진입 -> 단어 완료 토글 클릭 -> 실시간으로 해당 단어가 '암기 완료' 상태로 시각적 변경됨.

### 1.2 필수 기능 & UX 정의
- **파라미터 은닉화를 통한 주소 보호**: URL 경로에 `?chunk_id=700-marketing_1` 같은 파라미터를 그대로 노출하여 사용자가 주소를 마음대로 조작하거나 엉뚱한 리소스에 접근하는 위험을 원천 배제해야 합니다.
- **선택된 청크 동적 바인딩**: 컴포넌트가 마운트될 때 사용자의 프로필 상태에 있는 `selected` 속성을 감지하여, 해당 청크에 매핑된 단어 데이터를 0ms 수준으로 동적으로 렌더링해야 합니다.

### 1.3 구현 기술 및 데이터 매핑
- **URL 은닉화 및 selected 동적 타겟 매핑**:
  - 화면 라우터 설계 시 `/play` 및 `/quiz` 경로 뒤에 어떠한 동적 인덱스 파라미터도 붙이지 않도록 라우팅 규칙을 격리했습니다.
  - 리액트 훅 `useItem.jsx` 및 `Play.jsx` 컴포넌트 내부에서 `useMaster` 훅을 호출, 어떠한 인자도 넘기지 않는 기본 상태(`getChunkWords()`)로 트리거합니다.
  - 이 경우 훅 내부에서 `profile.selected` 에 지정된 고유 청크 식별자(예: `700-marketing_1`)를 자동으로 동적 캐치하여 로컬 캐시에서 해당 데이터를 파싱한 뒤 0ms 속도로 화면에 바인딩합니다.
- **관련 데이터 매핑**:
  - 단어 토글 시 로컬 캐시 `KEYS.VOCA` 객체 내의 해당 청크 `done` 배열에 단어 ID가 추가 또는 제외되며, 이는 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md#32-keysvoca-voca)의 데이터 규격을 따릅니다.

## 2. 학습 완료 시나리오 (Quiz & Chunk Completion)

### 2.1 시나리오 정의
- **상황**: 학습자가 오늘 지정된 단어 카드 40개를 모두 암기한 뒤, 제대로 외웠는지 점검하기 위해 퀴즈 풀기(`/quiz`) 단계를 수행하고 성공적으로 오늘 학습을 완료하는 시나리오입니다. 퀴즈를 모두 통과하는 시점에 해당 청크의 상태가 완벽히 '완료' 처리되어 통계에 집계되고 다음 청크로 자동 스왑되어야 합니다. 만약 현재 레벨의 모든 청크를 완수했다면, 자연스럽게 코스가 완수되었음을 알리고 수동으로 다음 난이도를 선택하도록 안내해야 합니다.
- **사용자 흐름**:
  - **일반 전진**: 퀴즈 풀기 진입 -> 40개 문제 풀이 성공 -> 결과 페이지 도달 -> 대시보드 홈으로 돌아왔을 때 오늘 학습량이 1 상승하고 다음 날 공부할 신규 청크가 암기하기 목표(`selected`)로 세팅되어 있음.
  - **레벨 전체 완수**: 마지막 남은 청크의 퀴즈 통과 -> 결과 페이지 도달 -> 대시보드 홈으로 이동 시 "초급 (700) 코스 완수 축하" 특별 배너와 설정 이동 링크가 표출되고 암기 목표(`selected`)는 공백(`""`) 상태로 비워져 있음.

### 2.2 필수 기능 & UX 정의
- **낙관적 상태 완료 마킹**: 사용자가 마지막 문제를 맞추고 결과창으로 넘어가는 즉시 화면 반응성을 보장하기 위해 로컬 캐시의 청크 상태 플래그(`status`)를 `true`로 즉시 마킹하고 완료 날짜(`completed_at`)를 오늘 날짜로 기록합니다.
- **다음 타겟 자동 전진 및 레벨 완수 예외 처리**: 
  - 퀴즈 완수 즉시 다음 날 학습할 최우선 스케줄 청크(`status = false` 인 첫 번째 순번)를 찾아내어 `profile.selected` 컬럼의 목표를 자동 전진시킵니다.
  - 만약 더 이상 미완료 청크가 존재하지 않는다면(코스 완수), `profile.selected`를 강제로 빈 문자열(`""`)로 설정하여 더 이상 자동 추천 단어가 대시보드에 뜨지 않도록 하고, 대시보드 배너에 추천 단어 카드 대신 레벨 완료 축하 배너 및 설정 바로가기 링크를 활성화합니다.

### 2.3 구현 기술 및 데이터 매핑
- **로컬 완료 마킹 및 동기화 인프라 호출**:
  - `src/app/pages/Quiz/Complete/Complete.jsx` 등에서 퀴즈 성공 이벤트 확인 시, `updateVoca(vocaLabel, doneList, true)` API 함수를 트리거합니다.
  - 로컬 엔진 `updateLocalVocaStatus` 가 구동되어 로컬 저장소 캐시 `KEYS.VOCA`와 `KEYS.PROFILE`에 `status = true`, `completed_at = YYYY-MM-DD`를 즉시 커밋합니다.
  - 이때 미완료 청크를 검색하여 있으면 해당 라벨을 `selected`로 갱신하고, 없으면 `selected = ""`로 비워줍니다.
  - 로컬 갱신 완료 즉시 공통 동기화 레이어의 백그라운드 프라미스를 이관 작동시켜 원격 Supabase DB 백업을 수행합니다. 원격 DB `User` 테이블의 `selected` 컬럼도 동일하게 빈 문자열(`""`)로 동기화됩니다. 구체적인 백그라운드 백업 동기화 기법은 [sync.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/sync.md)에 기술된 사양을 그대로 참조하여 상속 수행합니다.
- **관련 데이터 매핑**:
  - `Voca` 및 `User` 원격 테이블의 `status`, `completed_at`, `selected` 컬럼 값이 갱신되며, 상세 스키마 구조는 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md#1-supabase-원격-데이터베이스-테이블-명세)를 참조합니다.
