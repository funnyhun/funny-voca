# 단어장 도메인 명세서 (Vocabulary & Scheduling Domain Specification)

본 문서는 학습자의 목표 레벨에 따른 단어장 스케줄을 제공하고, 일정 변경 및 레벨 전입 시 학습 데이터를 안전하게 유지 및 조율하기 위한 사용자 중심 시나리오를 정의하며, 이를 완수하기 위해 설계된 시스템 기능 및 구체적인 기술 매핑 사양을 명세합니다.

---

## 1. 학습 데이터 제공 시나리오 (Data Loading)

### 1.1 시나리오 정의
- **상황**: 학습자가 본인의 실력 수준에 알맞은 영어 단어 개수를 제공받는 시나리오입니다. 학습 난이도를 변경함에 따라 단어장이 누적 범위로 자동 확장되어 알맞은 개수의 청크들이 스케줄로 잡혀야 합니다.
- **사용자 흐름**: 설정에서 레벨 800으로 변경 -> 대시보드로 이동 -> 기존 700 레벨 단어 1,200개에 더해 800 레벨 단어 1,200개가 추가로 결합되어 총 2,400개 단어(60청크)의 통합 학습 계획을 획득함.

### 1.2 필수 기능 & UX 정의
- **누적 범위 제공 사양**:
  - 700 레벨: 1,200개 단어 / 30청크
  - 800 레벨: 2,400개 단어 / 60청크
  - 900 레벨: 3,600개 단어 / 90청크
- **실시간 데이터 스케줄링**: 변경된 레벨의 모든 청크들은 권장 카테고리 순서 규칙에 맞춰 schedule 번호(1~N)가 실시간으로 촘촘히 재부여되어야 합니다.

### 1.3 구현 기술 및 데이터 매핑
- **lte 범위 쿼리 및 로컬 수식 변환**:
  - `src/api/voca/voca.local.js` 의 `getLocalVocaList` 에서 `[700, 800, 900].forEach((lvl)` 루프 내 분류 시, 단순 동등 비교에서 범위 이하 비교 수식(`Number(c.level) <= Number(lvl)`)으로 전환하여 하위 레벨 데이터를 누적 포함시킵니다.
  - 로컬 복구 및 API 연동 시, Supabase Chunk 테이블 조회 쿼리 체인에 `lte("level", targetLevel)` 조건을 강제하여 목표 레벨 이하의 전체 청크 데이터를 일괄 쿼리합니다.
- **관련 데이터 매핑**:
  - 조율된 레벨별 단어장 배열 형태는 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md#32-keysvoca-voca)의 `voca` 스키마 명세를 따릅니다.

## 2. 학습 데이터 변경 시나리오 (Data Re-scheduling)

### 2.1 시나리오 정의
- **상황**: 학습자가 더 먼저 암기하고 싶은 테마 카테고리를 학습 순서의 앞단으로 임의 스왑(Swap)하여 스케줄을 변경하거나, 공부하는 난이도를 변경(예: 700 -> 800)하는 시나리오입니다. 이 과정에서 기존 레벨에서 학습하며 획득한 암기 진도나 완료 기록이 소실되지 않고 완벽하게 상속 보존되어야 합니다.
- **사용자 흐름**:
  - 마케팅 청크와 일반오피스 청크의 순서를 바꿈 -> 정렬 순서가 실시간으로 교환됨.
  - 700레벨 학습 도중 일부 청크(예: 14개 청크)를 완수한 뒤 800레벨로 난이도를 높임 -> 14개 완료 이력이 800레벨 스케줄 내에서도 그대로 완료(status: true)로 남아 있음.

### 2.2 필수 기능 & UX 정의
- **진도 무손실 상속**: 레벨을 올리거나 스케줄을 재조정하더라도 이미 완료 처리한 개별 단어 정보(`done` 배열)와 청크 완수 여부(`status`)가 깨지지 않고 그대로 마이그레이션되어야 합니다.
- **DB 무결성 보장**: 원격 DB 백업 시 정렬 순번(`schedule`) 칼럼에 걸려 있는 복합 유니크 제약(Unique Constraint)으로 인한 데이터 유실이나 통신 에러가 없어야 합니다.

### 2.3 구현 기술 및 데이터 매핑
- **Object.values 딥카피 병합 상속**:
  - `rescheduleLocal` 에서 레벨 재배정 시 기존 학습 내역이 상속되도록 대조군 목록인 `filterOldList`를 단일 타겟 레벨 캐시가 아닌 전체 캐시를 병합한 형태(`Object.values(vocaList).flat().filter(Boolean)`)로 치환합니다.
  - `calculateNewSchedule` 함수 내부에서 `flatExisting.find` 대조 루프를 돌아 기존 완료 청크의 `done`, `status`, `completed_at` 값을 유실 없이 딥카피 및 바인딩해 상속시킵니다.
- **3단계 임시 음수 치환을 통한 유니크 충돌 우회**:
  - 사용자가 카테고리를 스왑하거나 레벨 마이그레이션을 단행하면, 원격 DB 동기화 모듈(`voca.sync.js`)이 가동됩니다.
  - 1단계: 순서를 바꿀 대상 행들의 `schedule` 칼럼 값을 임시 음수 일련번호(`-(i + 100)`)로 선제 업데이트하여 유니크 제약 공간에서 이탈시킵니다.
  - 2단계: 새로 부여될 양수 schedule 번호로 대상 행들을 순차적 업데이트하거나 신규 행을 Bulk Insert합니다.
  - 3단계: 음수로 임시 보관 중이던 행들의 schedule을 최종 목표 양수 정수값으로 치환하여 제약 충돌을 완벽히 우회합니다.
- **관련 데이터 매핑**:
  - `voca_label`, `schedule`, `done`, `status` 정보가 변경되며, DB 테이블 상세 명세는 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md#12-voca-사용자별-단어장-학습-상태-테이블)의 `Voca` 명세를 참고합니다.

## 3. 단어장 통합 레이아웃 및 다중 필터 검색 시나리오 (Voca Integrated Layout & Multi-filtering)

### 3.1 시나리오 정의
- **상황**: 학습자가 학습 달성 상태를 지속적으로 한눈에 파악하고 제어하면서, 전체 단어장 목록 또는 특정 단어장 내 단어들을 검색어와 다중 조건 필터를 통해 신속하게 찾아내고 학습하는 시나리오입니다.
- **사용자 흐름**:
  - 단어장(Voca) 페이지 진입 -> 상단 고정 헤더 영역에 오늘 학습할 청크 정보(Day 번호, 카테고리 한글명, 학습율 ProgressBar)가 실시간으로 보임.
  - 학습을 아직 끝내지 않은 경우 '학습하러 가기' 버튼을 눌러 즉시 시작하고, 오늘치 학습을 다 끝내서 100%가 된 경우 '완료' 뱃지 노출과 함께 '다시 학습하기' 버튼으로 변경되어 복습을 유도함.
  - 하단의 청크 목록 영역에서 레벨 필터('700', '800' 등)와 완료 여부 필터('완료', '미완료')를 다중으로 탭하고 검색창에 '채용'을 타이핑함 -> 조건에 완벽히 매칭되는 '채용' 관련 청크가 실시간으로 아래에 압축 노출됨.
  - 검색된 청크 상세 카드를 클릭하여 단어 상세 목록 페이지로 진입함 -> 상단 헤더 카드는 동일하게 유지되고, 검색과 필터 영역은 해당 청크 내 단어 명칭/뜻 검색 및 암기완료/미완료 단어 필터링(A-Z 정렬 등)으로 동적 스왑됨.
  - 단어 목록 상단 배너에 새로 추가된 뒤로가기 버튼을 누름 -> 즉시 이전 청크 목록 페이지로 복귀함.

### 3.2 필수 기능 & UX 정의
- **상시 노출형 학습 상태 대시보드 카드**:
  - 현재 활성화된 청크(`profile.selected` 키가 가리키는 청크)의 완료도(`done.length` / `word.length` 백분율)를 계산하여 상단 헤더에 상시 고정 표시합니다.
  - 달성도 수치에 따라 버튼 액션과 디자인이 동적으로 연동됩니다 (100% 미만: "학습하러 가기", 100% 완료: "다시 학습하기").
- **다중 조합 필터 및 실시간 라벨 검색 (청크 목록)**:
  - 레벨 필터(전체/700/800/900)와 상태 필터(전체/완료/미완료)를 각각 제공하며, 두 조건이 동시에 교집합으로 필터링되어야 합니다.
  - 검색어 입력 시 청크의 한국어 라벨(`category_kr`) 명칭 내 부분 일치를 기준으로 실시간 0ms 필터링이 적용됩니다.
- **단어 목록 내비게이션 복귀**:
  - 상세 단어 목록 진입 시, 사용자가 이전 청크 선택 상황으로 언제든 자유롭게 되돌아갈 수 있도록 시인성 높은 뒤로가기(Back) 버튼을 헤더 배너 내부에 배치합니다.

### 3.3 구현 기술 및 데이터 매핑
- **상태의 부모 리프팅(Lifting State Up) 및 Context 전파**:
  - `Voca.jsx`에서 `keyword`, `filterType`, `filterLevel`, `filterStatus` 상태를 중앙 선언하고 관리합니다.
  - 하위 라우트 렌더링 시 `<Outlet context={{ ... }} />` 구조를 통해 검색어와 필터 설정값 및 변경 메서드를 자식 컴포넌트(`VocaList`, `WordList`)에 다이렉트로 공유하여 동기화합니다.
- **다차원 메모리 필터링 알고리즘**:
  - `VocaList` (`List.jsx`)는 전달된 필터 상태들을 기반으로 다음과 같이 메모리 상에서 원본 청크 배열을 정밀 필터링합니다:
    1. `filterLevel`이 지정된 경우: `item.level === filterLevel` 조건 확인
    2. `filterStatus`가 'complete'인 경우 `item.status === true`, 'incomplete'인 경우 `!item.status` 조건 확인
    3. `keyword`가 존재할 경우: `item.category_kr` 부분 포함 여부 및 `schedule` 번호 매칭 여부 판별
  - `WordList` (`List.jsx`)는 단어 목록 배열에 대해 전달된 `keyword` 및 `filterType`을 적용하여 단어 단위 필터링을 수행합니다.
- **관련 데이터 매핑**:
  - 사용자 프로필의 선택된 학습 라벨(`profile.selected`) 및 각 청크의 고유 키(`voca_label`), 진행 상황 스펙은 [schema.md](file:///z:/home/minhulee/Projects/funny-voca/funny-voca-app/.agents/schema.md#12-voca-사용자별-단어장-학습-상태-테이블)의 `Voca` 및 `Profile` 스키마 명세를 정확히 유지합니다.

## 4. 학습 데이터 초기화 및 로그아웃 시나리오 (Data Reset & Logout)

### 4.1 시나리오 정의
- **상황**: 학습자가 기존 학습 진행 상황(진도, 단어 배정, Streak 정보 등)을 완전히 비우고 해당 레벨을 유지한 채 처음부터 새롭게 공부를 재생성하거나, 로그인 회원이 계정을 전환하기 위해 세션을 안전하게 종료하는 시나리오입니다. (게스트 사용자는 유저플로우상 로그아웃 기능이 제공되지 않으며, PWA 및 LocalStorage 기반 영속성 처리를 전적으로 신뢰하여 오작동을 방지합니다.)
- **사용자 흐름**:
  - **학습 데이터 초기화**: 설정 진입 -> '학습 데이터 초기화' 클릭 -> 2차 경고 다이얼로그 승인 -> 스피너 50% 진행 표시 -> 웰컴 온보딩 화면으로 강제 이동하여 해당 레벨 유지한 채 재생성.
  - **로그아웃 (회원 전용)**: 설정 진입 -> 최하단 '로그아웃' 클릭 -> 로그아웃 안내 다이얼로그 승인 -> 세션 스피너 활성화 -> 웰컴 온보딩 화면으로 안전하게 이동.

### 4.2 필수 기능 & UX 정의
- **이중 안전장치 컨펌**: 학습자의 실수로 인한 진행도 초기화를 완벽히 방지하기 위해 `window.confirm`을 통한 2단계의 경고 메시지를 필수로 통과하도록 구성합니다.
- **게스트 로그아웃 노출 제한**: 게스트 사용자는 모바일 PWA 환경에서 실수로 기기 영속 데이터가 전면 삭제되는 사고를 방지하기 위해 로그아웃 버튼 자체를 숨겨 유저플로우상에서 완전히 격리합니다.
- **데이터 서버 영구 보존 및 강제 소거 대응**: 로그인 회원의 학습 정보는 Supabase 원격 DB에 영구 백업 보존하고, 로그아웃 호출 시 세션 해제와 로컬 저장소 캐시만 깨끗하게 청소해 재로그인 시 복구될 수 있도록 조율합니다.
- **웰컴 화면 강제 리다이렉트**: 초기화 및 로그아웃 프로세스 완수 시점 즉시, 로컬 상태를 비우고 앱의 진입로인 온보딩 웰컴 페이지(`/welcome`)로 부드럽게 전이시킵니다.

### 4.3 구현 기술 및 데이터 매핑
- **로컬 스토리지 전면 청소 및 강제 전이**:
  - `useVoca` 훅 내의 `resetVoca()` 함수가 실행되면 API `deleteVoca()`가 구동됩니다.
  - `deleteLocalVoca()` 및 `clearAllCache()`를 순차 트리거하여 로컬 브라우저 캐시인 `KEYS.VOCA`, `KEYS.PROFILE` 등의 모든 데이터를 일괄 삭제합니다.
  - 리액트 상태가 비워진 후 `window.location.href = "/welcome"`을 통해 브라우저 수준에서 온보딩 페이지로 강제 리다이렉트시킵니다.
- **넌블로킹 백그라운드 원격 삭제 위임**:
  - API `deleteVoca()` 내부에서 로그인 회원으로 판별된 경우, `await`로 메인 렌더링 스레드를 블로킹하지 않고 백그라운드 비동기 프로미스를 호출하여 `deleteRemoteVoca(session.user.id)`를 실행합니다.
  - `deleteRemoteVoca` 함수는 Supabase `Voca` 테이블의 모든 기록을 `delete()` 처리하고 `User` 테이블의 진행도 컬럼(`selected`를 빈 문자열 `""`, `completed_date`를 `null`)을 초기화(`update()`)합니다.
- **회원 인증 세션 해제**:
  - `logout()` 호출 시 `supabase.auth.signOut()`을 백그라운드로 실행하여 서버 측 세션을 해제하고, 로컬 캐시를 일괄 지운 뒤 리액트 라우터의 `navigate("/welcome")`를 사용하여 전이시킵니다.
- **관련 데이터 매핑**:
